# CHARACTERS_AND_SKILLS.md — 人物数值 · 队友 · 武学设定

> 数值与养成的单一事实来源，与 STORY_BIBLE.md（剧情）、GAME_DESIGN.md（系统）配套。
> 实施时数值以此为准填入 `src/data/characters|skills/`；平衡性调整改这里（同 PR）。
> 原则：**数字少而可读**（6 项属性）、**成长重武学轻等级**（继承原版"练功"手感）、
> **每个队友一个不可替代的定位**（不做数值复读机）。

---

## 1. 属性模型（6 项，M2 起生效）

| 属性    | 作用                   | 主角初始 |
| ------- | ---------------------- | -------- |
| hp      | 生命，0 = 倒下         | 50       |
| mp      | 内力，施展武学消耗     | 20       |
| attack  | 攻击，进伤害公式       | 10       |
| defense | 防御，进伤害公式       | 5        |
| speed   | 速度，决定行动顺序     | 10       |
| move    | 移动力，每回合可走格数 | 4        |

伤害公式（NEXT_STEPS §2.2 已定）：
`damage = max(1, round((attack + skill.power - defense) * counterMod * variance))`
克制环 刚→奇→柔→刚（1.25 / 0.75），variance = 0.9 + rng.next() × 0.2。

## 2. 成长模型

### 2.1 主角：历练等级（轻）+ 武学熟练度（重）

- **历练**：战斗/剧情事件给历练值，升级曲线 `所需 = 100 × lv^1.5`，上限 30 级。
  每级：hp +8、mp +4、attack +2、defense +1、speed 每 2 级 +1、move 固定 4
  （move 只靠轻功类武学提升——移动力是设计资源，不通胀）。
- **武学熟练度**：每门武学 1-10 级，**用一次涨一点熟练**（曲线 `升级所需 = 20 × lv`），
  每级 power +10%。读秘籍 = 解锁武学 + 直升 3 级（原版"读书练功"的手感）。
- 30 级满级主角裸装约：hp 290 / mp 136 / atk 68 / def 34 / spd 24 —— 所有敌人数值
  以此锚定。

### 2.2 队友：跟随成长（不刷级）

队友加入时等级 = 主角等级，之后随主角同步升级（免二队练级之苦——现代化取舍，
记入 GAME_DESIGN §2 表）。个体差异全部体现在**成长系数**与**专属武学**上：

`队友属性 = 主角同级基准 × 角色系数`（系数见 §4 表）。

### 2.3 敌人分层（对应 STORY_BIBLE §1.2 难度层）

| 层      | 相当于主角等级 | 代表                           |
| ------- | -------------- | ------------------------------ |
| T1 杂兵 | 1-5            | 太岳四侠、吕梁三杰             |
| T1 BOSS | 6-8            | 卓天雄                         |
| T2 杂兵 | 8-14           | 万府家丁、温家堡弟子           |
| T2 BOSS | 14-18          | 血刀老祖、张召重               |
| T3 杂兵 | 16-24          | 蒙古兵、六大派/魔教弟子        |
| T3 BOSS | 24-30          | 欧阳锋、金轮法王、成昆         |
| 天花板  | 30+（超模）    | 东方不败（spd 40，必须战术破） |

## 3. 武学设定

### 3.1 分类

- **外功**（主动，占行动）：刚 / 柔 / 奇 三系 + 无系普攻。字段：school, power, range,
  area(单体/十字/直线), mpCost。
- **轻功**（被动）：+move / +speed。M4 实装，一人一门。
- **内功**（被动槽，一人一门）：全属性百分比加成或特殊效果。M4 实装。

### 4.1 羁绊系统挂点（M4）

每名队友 1-3 个羁绊事件（数据化：flag 链），满羁绊解锁专属武学升级或结局条件
（〔逍遥归隐〕）。标杆实现：令狐冲×任盈盈（同队触发链）、杨过×小龙女（双剑合璧）、
郭靖（降龙传授）。

## 5. 正邪值 · 声望的数值挂钩（M4，先定规则）

- 正邪值 = 单轴 [-100, +100]，抉择偏移 ±5（小）/ ±15（章节级）/ ±30（线级站边）。
  界面按 GAME_DESIGN §2 半明示：只显示称号档位
  （大侠 ≥60 / 侠士 ≥20 / 无名 / 枭雄 ≤-20 / 魔头 ≤-60）。
- 招募门槛示例：何铁手 邪≤-20；降龙十八掌 侠≥60；黑木崖邪线入口 邪≤-40。
- 门派声望独立于正邪（可以是邪值很高但少林声望不低的"真小人"），每派 [-50, +50]，
  影响拜师武学与部分对话变体。

## 6. 实施顺序（与 NEXT_STEPS / STORY_BIBLE 对齐）

1. **M2**：属性模型 §1 + 伤害公式 + 敌人 T1 数值（强盗=T1 杂兵模板）。武学只需
   野球拳 + 1-2 门测试用刚/柔技（数值从 §3.2 取）。
2. **M3**：历练等级 §2.1、武学熟练度、消耗品 §3.3；郭靖黄蓉按 §4 系数实装（先作
   剧情战友军，招募系统 M4 才上）。
3. **M4**：队友名册批量实装 + 羁绊挂点 + 正邪/声望数值 §5 + 轻功/内功被动槽。
4. 平衡性验证方式：每层写一个"标准队打标准 BOSS"的模拟测试
   （`src/game/battle/__tests__/balance.test.ts`，固定 seed 跑 resolve 序列断言回合数
   落在设计区间 4-8 回合）——数值改坏了测试会叫。
