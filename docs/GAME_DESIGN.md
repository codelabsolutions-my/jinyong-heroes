# GAME_DESIGN.md — 游戏设计文档

> 系统与内容设计的单一事实来源。数值、剧情结构、系统规则改动都更新这里（同 PR）。
> 定位：《金庸群侠传》(1996, 河洛工作室) 的**现代化重制** — 保留原版的剧情骨架和
> 开放式结构，用现代游戏设计重做体验，并扩展新剧情与新元素。同人学习项目，非商业。
>
> 配套细化文档：**剧情**（十四线章节流程、结局矩阵）→ `STORY_BIBLE.md`；
> **数值**（属性/成长/队友/武学表）→ `CHARACTERS_AND_SKILLS.md`。三份冲突时，
> 本文档管"系统规则"，那两份管各自领域的细节。

## 1. 核心循环（继承自原版）

```
探索大地图 → 触发各部小说剧情 → 战斗/抉择 → 学武功、招队友、收天书
    ↑                                                        ↓
    ←←←←←←←  变强后解锁更多区域与剧情，最终集齐十四天书  ←←←←←←←
```

- 主角**小虾米**穿越进金庸武侠世界，需集齐十四部天书（"飞雪连天射白鹿，笑书神侠倚碧鸳"）回到现实。
- **开放世界**：除少数门槛外，玩家可按任意顺序探索、触发各小说的剧情线。
- **正邪值**：行为影响道德立场，决定可招募的队友、可学的武学与结局分支。
- **招募**：可将乔峰、令狐冲、张无忌等原著人物纳入队伍（上限 5 人同战）。

## 2. 原版系统 → 现代化处理

| 系统     | 原版                   | 重制处理                                                                |
| -------- | ---------------------- | ----------------------------------------------------------------------- |
| 战斗     | 回合制战棋（网格走位） | 保留战棋核心；加：速度决定行动序、地形效果、技能范围预览、可撤退        |
| 移动     | 逐格移动               | 保留逐格手感，插值动画（已实现）                                        |
| 武学     | 秘籍研读 + 使用升级    | 保留；加：武学树 UI、属性克制（刚/柔/奇）明示化                         |
| 正邪值   | 隐藏数值               | 半明示：给玩家"江湖名声"面板，具体数值仍隐藏                            |
| 任务     | 无任何指引，靠背攻略   | **任务日志 + 线索系统**：记录已听到的传闻，不给箭头不给坐标，保留探索感 |
| 存档     | 客栈存档               | 随时存档 + 多档位；铁人模式（可选）致敬原版                             |
| 死亡惩罚 | 团灭读档               | 战败可被俘/被救的剧情分支（部分战斗），减少读档挫败                     |
| 练级     | 野球拳刷级             | 保留野球拳彩蛋；数值曲线重做，减少纯刷                                  |

## 3. 新增元素（本重制的差异化）

### 3.1 扩展剧情

- **原版每部小说只取 1-2 个片段**；重制为每部小说做成 2-4 章的迷你剧情线，有开端-转折-收束。
- **人物羁绊事件**：队友之间/队友与主角之间的支线（如让令狐冲与任盈盈同队触发额外事件）。
- **反派线**：正邪值走邪路时有完整的邪派剧情，不只是"少了正派选项"。
- **新增原创角色**：仅限配角/引导者定位，不改写原著人物性格。

### 3.2 新系统

- **传闻/线索系统**：NPC 对话产出"线索卡"，任务日志按小说分册整理 — 这是无箭头开放世界的可玩性支柱。
- **声望分面**：不止一条正邪轴 — 各大门派有独立好感（少林、武当、日月神教…），影响拜师与剧情。
- **多结局**：按天书收集度 × 正邪值 × 关键抉择，至少 4 个结局。
- **活江湖（M6）**：门派动态模拟（确定性 tick 产世界事件→传闻）+ 自创武功
  （点数预算）— 设计见 `AI_FEATURES.md`。

### 3.3 明确不做（scope 控制）

- 不做实时/动作战斗；不做多人；不做配音。
- 不做**运行时** LLM / 后端（ADR #11，AI_FEATURES.md）——AI 只用于创作期内容生产。
- "随机生成内容"限定为：不做程序化地图/任务；门派模拟的世界事件是确定性规则 +
  注入 RNG，可回放可测试，不属于此列。

## 4. 战斗系统规格（M2 已实现，见 `src/game/battle/`）

- 网格战棋（曼哈顿 / 四连通，与探索移动模型一致）；每回合按 speed 降序行动，
  同速用稳定次序，死亡单位不入序（`turnOrder.ts`）。
- 一个回合 = 移动（≤move 步 BFS，障碍/单位阻挡）+ 一个动作（攻击 | 武学 | 待机）。
  **物品推迟到 M3**（随消耗品/商店一起，见 CHARACTERS_AND_SKILLS §3.3）。
- 伤害 `max(1, round((atk+power-def) × counterMod × variance))`；三系克制环
  刚→奇→柔→刚（克制 ×1.25 / 被克 ×0.75 / 同系无系 ×1.0）；variance = 0.9+rng×0.2。
- 结算核心 `resolve(state, action, rng)` 是纯函数（CLAUDE.md §5.5），RNG 显式注入、
  可回放（同 seed+同序列→同结果，有回放单测）。战场/单位/武学全部收敛进自包含的
  `BattleState`，`setup.ts` 是唯一的 data→state 翻译点。
- **M2 简化项**（M4 前补齐）：单位无常驻系别，防守方按无系结算；移动一经确认不可撤销
  （无 undo）；敌方 AI 为确定性趋近-攻击；战斗不进存档，战后满血（不 bump SAVE_VERSION）。
- **战败处理（M2）**：退回探索原地、**保留全部进度**（胜利 flag 未置，可再挑战）。
  不做"团灭读档/死亡惩罚"——那需要持久队伍状态与检查点设计，留到 M3+（见 ADR #16）。

## 5. 数据驱动的内容格式

全部内容在 `src/data/` 下（CLAUDE.md §5.2）：

- `maps/` — 地图（字符网格 + 地形表 + 出入口 + NPC 摆放）
- `characters/` — 人物：属性、武学、招募条件、羁绊
- `skills/` — 武学：系别、威力、范围、升级曲线
- `story/` — 剧情事件：`{ id, trigger: Condition, steps: [dialogue|choice|battle|reward|setFlag] }`
- `books/` — 十四天书的获取路径定义

`Condition` 是声明式的：`{ flag, minMorality, hasCompanion, bookCount, … }` 组合，由 ConditionEvaluator 统一求值。
