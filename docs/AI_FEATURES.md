## 1. 创作期 LLM 工作流（随时可用，不等 M6）

LLM（任何模型，包括写内容的 Claude session 本身）用来批量生产：

- **环境闲聊池**：每个 NPC 20-50 条按条件变化的台词（剧情进度/正邪称号/所在线完成度），
  格式就是现有的对话变体——只是量大，所以用生成。
- **传闻文本池**：给 §2 的门派模拟事件配叙述模板（每类事件 5-10 种说法）。
- **对话初稿**：STORY_BIBLE 章节 → 对话脚本初稿，人工润色后入库。

**流程铁律**：生成物必须过 ①`content.test.ts`（坏链/遮蔽检查自动拦）②人工读一遍
（人物像不像原著——写作规范见 STORY_BIBLE §5）③正常 PR。生成 ≠ 免审。

## 2. 门派动态模拟（确定性，无 LLM）

"江湖在你不在的地方活着"——用模拟层实现，不用生成：

- `src/game/world/factionSim.ts`：门派关系矩阵（每对门派 [-100,100]）+ 事件 tick。
  玩家每完成一章/过一定游戏内天数，tick 一次：按关系值+RNG（注入，可回放）产出
  世界事件（摩擦/结盟/比武/商路遇袭…），事件改关系值，也受玩家行为影响
  （声望、站边抉择直接进矩阵）。
- 世界事件 → 查**传闻文本池**（§1 预生成）→ 进日志"传闻"分册，茶馆/太岳四侠处可听。
- 部分事件开放介入钩子（声明式条件复用 Condition 系统）：介入改变关系走向。
- 全部状态进 GameState（需 bump SAVE_VERSION）；tick 是纯函数，同 seed 可回放，
  单测断言"关系矩阵演化轨迹"。

## 3. 自创武功（确定性点数预算，无 LLM）

- **数值**：固定预算点数分配（power/range/area/mpCost 互相抵扣，预算随主角等级/
  熟练度成长解锁），公式在 `src/game/skills/create.ts`，单测穷举边界防超模。
  三系选择影响克制关系，预算总量三系一致。
- **名字与描述**：组合词库（前缀〔玄冥/流云/追风/罗汉/…〕×主体〔掌/剑/指/腿/功〕×
  意境短语库），词库创作期生成（§1），运行时确定性拼装。玩家也可手动输入名字
  （长度/字符白名单校验）。
- 定位：M6 的养成彩蛋，不参与主线平衡锚定（BOSS 数值仍按 CHARACTERS_AND_SKILLS §2.3）。
